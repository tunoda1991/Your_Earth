Known Issues & Architecture Gaps

  • 🔴 make-server Edge Function: auth profile, signup, communities, discussions, connections, messages, notifications all depend on an external Figma Make function CRITICAL
  • 🔴 5 missing Supabase tables: user_settings, community_members, event_attendees, user_activity, events (native) CRITICAL
  • 🔴 Two separate feed systems: Express /api/feed/* (Drizzle) vs Supabase posts table (different schemas, not synced) CRITICAL
  • 🔴 Disaster card photos (d1.jpg–d178.jpg) missing from client/public/ CRITICAL
  • 🟡 No real auth on Express feed API: X-User-Name header only, trivially spoofable WARNING
  • 🟡 Navigation search bar is non-functional (console.log stub) WARNING
  • 🟡 Notifications panel: 5 hardcoded fake notifications, no backend WARNING
  • 🟡 LiveImpactTicker: static fake activity, not connected to real data WARNING
  • 🟡 Jobs & Marketplace use DUMMY_* arrays with no API backend WARNING
  • 🟡 Quick Login bypasses Supabase auth with fake user objects WARNING
  • 🔵 favicon.svg referenced in index.html but only favicon.png exists INFO
  • 🔵 Dead dependencies: passport, express-session, connect-pg-simple, memorystore never imported INFO

Architecture Overview

Frontend
React 19 + Vite 7 + Tailwind v4
Routing: ?page= query param + pushState
State: TanStack Query + React state
Server
Express 5 + Drizzle ORM
Feed API: /api/feed/*
Proxy: /api/electricity-maps/*
Database
Supabase PostgreSQL (primary)
Drizzle/MemStorage (feed fallback)
DATABASE_URL toggles persistence
Auth
Supabase Auth (signIn/signOut)
make-server (signUp/profile)
Express: X-User-Name header
Edge Functions
curate-news – RSS + Gemini AI
curate-social – Reddit/Mastodon
Requires: GEMINI_API_KEY
Maps
Standalone HTML + WebGL/Canvas
Embedded via <iframe>
